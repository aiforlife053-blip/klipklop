## General Rules

1. Do not overthink. Choose the simplest solution that satisfies the requirements. Avoid unnecessary complexity or premature optimization.

2. Before starting any task, identify and use only the relevant skills and MCPs. Do not invoke unrelated skills or tools.

3. Before reading project files, understand the repository structure using the existing CodeGraph database (`.codegraph/codegraph.db`). Use CodeGraph as the primary navigation tool, then read only the files required for the current task. Do not scan the entire repository unless absolutely necessary.

4. Complete milestones strictly in order. After finishing a milestone, stop and wait for my approval before starting the next one. Do not implement features or tasks from future milestones in advance.
