Key Decisions
- Download source once (not per-clip section) → fewer yt-dlp requests, faster
- face_tracking_mode="center" default → avoids OpenCV haarcascade crash on Windows
- gemini-2.5-flash replaces gemini-3-flash-preview (model doesn't exist)
- Blur background = animated (follows video per frame), not static
- Blur toggle default = OFF (crop faster; user enables for landscape)
- Portrait output resolution tied to quality, not hardcoded 1080x1920
- Save = union of saved_clips (not overwrite), so multiple save actions accumulate
- Delete master.mp4 → deletes parent clip folder entirely

- Prompt filter: 45 <= duration <= 75 accepted; AI prompt says 50–70s target
- saved_clips in run.json: list of absolute paths to saved master.mp4 files
- Log deduplication: compares last entry message only
- _format_selector() builds yt-dlp format string from self.video_quality
- video_quality validated to {"480","720","1080"}, defaults "720"
- output_resolution map: "480"→"540:960", "720"→"720:1280", "1080"→"1080:1920"
